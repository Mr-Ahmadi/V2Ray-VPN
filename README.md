# V2Ray VPN

Desktop V2Ray client built with Electron and React.

![V2Ray VPN Client Screenshot](Screenshot.png)

## Features

- **Server management** — VLESS, VMess, Trojan, Shadowsocks
- **Proxy modes** — Global (system proxy), Per-app (selected apps), PAC (auto-config)
- **Application routing** — Per-app VPN/bypass policies for Chromium-based browsers (Chrome, Edge, Brave, Opera, Vivaldi, Arc)
- **DNS management** — Provider presets, custom DNS profiles, apply to system
- **Live traffic stats** — Real-time upload/download and ping monitoring
- **Kill switch** — Block traffic on VPN disconnect
- **Auto-connect & reconnect** — Launch on startup, reconnect on drop
- **Update checker** — GitHub Releases integration for updates

## Requirements

- Node.js 18+
- npm 9+
- V2Ray / Xray core binary (see Setup below)

## Setup

Run the setup script to download the Xray core:

```bash
./setup.sh
```

This downloads the latest Xray release, extracts it to `v2ray-core/`, and renames the binary to `v2ray` (the app expects this name).

## Development

```bash
npm install
npm run dev
```

Starts the React dev server (port 3000) and launches the Electron app.

## Build

```bash
npm run build     # compile renderer + main
npm run dist      # build + package installers (macOS DMG)
npm run dist:github  # build + upload to GitHub Releases
```

## Project structure

```
src/
  main/           # Electron main process
  renderer/       # React UI (CRA)
  services/       # Core services (V2Ray, routing, DNS, proxy)
  db/             # SQLite database layer
  types/          # TypeScript declarations
v2ray-core/       # Xray binary + geo data (generated by setup.sh)
```

## Tech stack

- Electron — desktop shell
- React + TypeScript — UI
- Material UI — component library
- SQLite (better-sqlite3) — settings & server persistence
- Xray-core — proxy engine
